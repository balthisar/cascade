Preferences allow you to customize some of the aspects of how Cascade operates. The preferences panel is also the means to identify the author of Cascade documents and the stylesheets it generates. The following illustrations show both pages of the preferences panel, and below them, the options are illustrated and explained.

Preferences Page 1



Page one of the preferences panel displays the preferences proper. The items in this panel function as follows:

 Determines whether or not the Font-Family drop-down lists display the currently selected font in their own typeface.

 Determines whether or not the combo editors for color values display the currently chosen color. If not, only the name or hexadecimal value of the color is display (in any case, system colors will never be displayed in thier own color).

 Determines whether or not the HTML Color Picker will allow you the option of selecting system colors.

 Determines whether or not Cascade confirms that you really want to cut or delete the currently selected class in the class list.

 Cascade would normally use double-quotes when exporting attributes (e.g., "Times New Roman). Sometimes this conflicts with existing HTML, particularly if you are placing CSS inline. Checking this option tells Cascade to use single-quotes when exporting attributes (e.g., 'Times New Roman').

 Determines whether or not Cascade allows you to insert more than one selector with the same name. Older versions of Cascade would try to "protect" the user from doing this, but with the introduction of @media types, such "protection" is no longer desireable all of the time. Besides, allowing multiple instances of the same selector provides other flexibilities in a CSS2 stylesheet.

 Specifies the default Author Name, Author Email, and Author URL for new Cascade documents. This information will be included in any Cascade documents you create.

 Allows you to specify the default drive and path where Cascade first attempts to open or save documents. After the initial open or save operation, Cascade will use the last folder you accessed until the next time you start the application (in which case, the default is used again initially).


Preferences Page 2



 Page two of the preferences panel displays the current document information, including the original author's name, email address, and URL if he specified them. Also shown is the name of the last application which saved the document, as well as any document special features. This last item is reserved for future expansion of the CSS2 file format.
The following illustration shows the "Apply Identity" button that is on page two of the preferences, and it is described below.

 Cascade normally leaves the original author's information in the document as a constant reminder of the document origin. However, if you make significant changes to a document, or need to change the name, email address, or URL of your own documents, then this button will "steal" the information from the default author items on page one of the preferences, and apply it to the current document. Voila! You are now the original author of the document.
