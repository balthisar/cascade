 Unfortunately, a "Class" in Cascade refers to two different things:
  • Any CSS2 class in general, as described in the overview. For example, an "Element class" is a type of class.
  • And as described below, a specific type of class, called simply enough, a "class" is a type of class in its own right.

    In CSS2 and Cascade, the class of type "Class" can be applied to any element at any time. Its selector takes the form ".classname" (take note of the initial period or dot). Whenever an HTML element uses a CLASS attribute with the value set to the class name, the class' attributes affect the element's rendering. For example, if...

    the stylesheet has:
    .classname {font-family: "Helvetica"}
     and the HTML has:
    <P CLASS=classname>

     ...then the <P> element will display with all of its normal CSS attributes, except for those which have been overridden in the class .classname; in the above example, the <P> element would be rendered using the font "Helvetica" instead of <P>'s default font. Notice that the <P> element's CLASS attribute does not include the initial period of the class name.

    To create new element classes in Cascade, use the Classes/New Class... command in the     main menu.
