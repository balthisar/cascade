(As Implemented by Balthisar Cascade)

 Balthisar Cascade makes it simple to create a variety of the basic classes as defined in the W3C's specification document. It also permits the advanced user to create classes having any selector, so that he doesn't have to be hampered by the implementation of basic classes.

This help section is intended to provide a brief background of the types of classes you can create in Cascade. It is not intended to explain how to successfully use any type of class, nor how to create valid selectors for more advanced classes.

The first thing to know is, what exactly is a class (in the general sense)? In CSS2 and in Cascade, a class is a style rule that governs how a certain element is rendered on the screen, on the printed page, or aurally. A class consists of certain named attributes, each of which can have different values. For example, "Font-Familiy" is an attribute of any class, and it can have a value such as "Helvetica."
Additionally, each class has a name or label which distinguishes the class from all other classes. This label allows the user agent to select the proper attributes to match the on-screen element. Hence, this name or label is properly known as a "selector."
Cascade's class list shows a list of selectors and media descriptors. The attributes of each class are set in the attributes panes. The CSS2 code is shown in the preview pane. The selectors in the class list will often be referred to as a class, because the selector represents the entire class.

Cascade and the CSS2 specification recognize different types of classes, which behave in different ways in the user agent. Cascade makes using the basic classes as fool-proof as possible, and also provides the means for more experienced users to create any type of advanced selector. All types of classes are identical to each other, except for the format of the selectors, and how the classes are implemented.
