Contextual Classes

Table of Contents Balthisar Software Site Cascade Classes


 Contextual classes make it possible to tell the user agent that certain CSS2 style rules should be applied to Element classes only if they are preceded by particular other elements. For example, suppose you are content to leave the default styles for the <P> HTML Element alone, except when the <P> element occurs in a <TABLE> tagset. One option could be just to create a subclass of <P>, and remember to use it inside of any tables you may have on your page. Another alternative is to create a contextual selector, as shown in the example:

the stylesheet has:
TABLE P {font-family: "Helvetica"}

 and the HTML has:
<P>Hello there
<TABLE ROWS=1 COLS=1 WIDTH=50%>
<TD><P>Sentence in a table</TD>
</TABLE>

 If we were to see the example in the user agent, the paragraph "Hello there" would be rendered using the default style for content of a <P> element. However, the paragraph "Sentence in a table" would be rendered in the "Helvetica" font, because the <P> element occurred nested somewhere within a <TABLE> tagset.

They type of contextual selector implemented is called a descendant selector. There are also other types of contextual selectors supported by the CSS2 specification, and you can create these by creating a Custom/Expert Class. The CSS2 Specification from W3C examplifies the other types of contextual selectors.

To create new contextual class in Cascade, make sure that an Element class is selected in the class list. This should be the nesting element, i.e., the element tagset inside of which your contextual class is valid. Then, execute the Classes/Contextual Class... command from the main menu, or select Contextual Class... in the class list's contextual menu. If you wish to create a contextual class for which there is not already an element in the class list, you must use use a Custom/Expert Class*.

*Actually, there are two "holes" in the program that will allow you to create a Contextual Class without having the context-element in the list, and without having to create a Custom/Expert class. Any class of any type which is dragged-and-dropped from one document to another, and the selector doesn't already exist in the destination document, will be added to the destination document. Also, it's possible to create an Element class temporarily, create a contextual class, and delete the temporary Element class.
