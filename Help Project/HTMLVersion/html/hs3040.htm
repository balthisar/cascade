Pseudo-Classes and Pseudo-Elements

Table of Contents Balthisar Software Site Cascade Classes


 Pseudo-classes and pseudo-elements permit the creation of classes that apply to classes, subclasses, and element classes in somewhat dynamic situations. For example, in your own experience in using web browsers, your <A> elements usually display in a different color, depending upon whether you have visited the link or not. This is an example of a dynamic situation; the same <A> element class acts as if it has a different "Color"attribute value, depending on whether or not the link has been visited. This is accomplished using Pseudo-classes. For example...

the stylesheet has:
A:visited {color: red}
A:link {color: blue}

 and the HTML has:
<A HREF="somewhere">Link to Somewhere</A>

 ... then depending on whether the link has been visited or not, the user agent will dynamically select which pseudo-class to apply to the content of the <A> tagset.
As you should have noticed, the selector for a pseudo-class or pseudo-element takes the form of "ELEMENT:pseudo" (where ELEMENT is an HTML element).
To create new pseudo-class or pseudo-element in Cascade, make sure that an Element class is selected in the class list, and execute the Classes/Pseudoclass... command from the main menu, or select Pseudoclass... in the class list's contextual menu.
