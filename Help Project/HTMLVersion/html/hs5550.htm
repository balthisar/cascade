 If you have a stationary file that resides in the same directory as the Cascade application, and it is named "default.WWTD" then in most cases, Cascade will create new documents based on that file instead of creating empty documents.
If the above conditions are met, and you use the File/New... command in the main menu, or use the new file shortcut, the new document will be based on "default.WWTD."
If you use the Ctrl-N keypress, then Cascade will create the new document with no classes at all (i.e., it will ignore the presense of "default.WWTD").
