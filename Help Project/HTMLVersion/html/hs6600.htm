Including CSS in HTML

Table of Contents Balthisar Software Site Practical Use


 Balthisar Cascade can automatically change any HTML documents that you select so that they will work with your cascading style sheets. This functionality is achieved using the Tools/Apply Stylesheet to HTML File(s)... command on the main menu.
However, this Cascade feature may fail under certain circumstances, or, you may choose not to use the feature. Therefore, this section explains how you can alter your HTML documents manually.

The CSS2 specification and HTML 3.2+ are very liberal in how the HTML author can use a cascading stylesheet.
There are two varieties of using style sheets in an HTML document, with four distinct methods. The variety you wish to use will determine how you should export your Cascade document (note: a Cascade project file, a file with a .WWS2 extension, is only useful for Balthisar Cascade and other applications which support the file format. For use in HTML documents, it is necessary to use one of Cascade's export functions).
The simplest and perhaps most useful variety of using your stylesheets involves linking to an external file which contains your stylesheet rules. This file is generated by Cascade when you select CSS as your filetype when exporting your Cascade project. This provides the benefit of requiring less stylesheet information if all of your pages are using a similar look-and-feel. It also eliminates editing individual HTML documents, if all of your style information is in a single file. Two methods exist to accomplish this external linking.

One Method:

 <STYLE TYPE="text/css">
<!--
@import url(url-to-your-file.css)
-->
</STYLE>

 An additional benefit of the above method is that you can define or override rules from the imported stylesheet within the same <STYLE> tagset. The user agent treats the imported style information as if it were included in the HTML document.


Alternatively:

 <LINK REL=STYLESHEET TYPE="text/css" HREF="your-url" TITLE="any">

If you don't intend to override rules in the imported stylesheet, the above method is supurb. This is the method Cascade uses if you instruct it to apply stylesheets to your HTML documents.


The second variety of including style information into your HTML document is to simply embed it into your HTML source code. This saves the user agent from having to open a separate connection to the server in order to get style information. On the down-side, if you have a look-and-feel which is consistant across all of your pages, a change to a class will involve editing many files.
This variety is probably best suited for defining classes which appear in only one HTML document.
By selecting HTML as your filetype, you can simply use your editor's import function, or copy and paste the file into your HTML document.

One Method:

 <STYLE>
<!--
A {
font-family: "Times New Roman"; {etc}
 }
-->
</STYLE>

 The <!-- and --> tags within the <STYLE> elements serve as comments, so that user agents not able parse <STYLE> tags will not display your style sheet rules as on-screen text.

Alternatively:

 <P STYLE="color:yellow; font-family:Chicago">
This is an HTML Paragraph
</P>

 The above method is suited for instances in which you desire a particular style for a single instant only. Frequent use of the above method makes HTML documents difficult to read and modify.
